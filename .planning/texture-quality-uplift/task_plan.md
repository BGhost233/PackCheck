# 质感跃迁施工计划 (Texture Quality Uplift)

## 目标
在不影响当前已实现功能的前提下，通过低成本高回报的视觉细节改动，将 PackCheck 的界面质感从「功能完整」提升到「值得截图」。

## 约束
- 不改变 HdsTabs + hdsMaterial 沉浸光感体系
- 不改变现有数据结构（向后兼容）
- 不改变组件层级结构和数据流
- 每项改动独立可回滚（细粒度 commit）
- 每项改动后必须 build 验证通过

---

## Phase 1: 一行代码级提升（零风险，改动 < 20 行/项）
**状态：** `pending`

### 1.1 fontFeature tnum 等宽数字
- **改动点：** DesignTokens 新增常量 + WeightGauge / HomePage 倒计时 / 清单进度数字加 `.fontFeature("'tnum'")`
- **影响范围：** 纯视觉，不影响布局逻辑（等宽数字可能微调数字宽度，需确认不溢出容器）
- **风险：** ⚠️ 低风险——HarmonyOS Sans 确认支持 tnum，但需验证 fontFeature 参数格式（单引号包裹的 `"'tnum'"` vs 不包裹的 `"tnum"`）
- **回滚：** 删除 `.fontFeature()` 行即可

### 1.2 卡片底部微边框
- **改动点：** 全局卡片样式（DesignTokens 定义 + 各卡片组件）加 `.border({ width: { bottom: 0.5 }, color: { bottom: '#E8E8E8' } })`
- **影响范围：** 纯视觉，0.5vp 不增加任何布局尺寸（border 默认在 padding 内）
- **风险：** ⚠️ 极低——需确认 ArkUI 是否支持 0.5vp width（不支持则用 1）。卡片如已有 `.border()` 设置会覆盖而非叠加
- **回滚：** 删除 `.border()` 行

### 1.3 错落动画加速度曲线
- **改动点：** AnimationTokens 中的 stagger delay 计算公式从 `index * 40` 改为 `index * 30 + index * index * 4`
- **影响范围：** 列表入场动画节奏变化（首项更快、末项更慢）
- **风险：** ⚠️ 低风险——长列表（>10 项）末尾项延迟会显著增大（第 10 项 = 700ms），需设上限 cap
- **缓解：** 公式加 `Math.min(cap, ...)` 封顶，cap = 400ms
- **回滚：** 改回 `index * 40`

### 1.4 长按编辑态微倾斜
- **改动点：** 装备卡片组件，进入可操作态时加 `.rotate({ angle: randomAngle })` + `.animation(spring)`
- **影响范围：** 仅装备库中需要长按操作的卡片（如果有的话）
- **风险：** ⚠️ 需确认当前是否有长按编辑态。如果没有该功能则此项延后
- **回滚：** 删除 `.rotate()` 行

## Phase 2: 小模块级提升（30-80 行/项，低风险）
**状态：** `pending`

### 2.1 噪点纹理背景
- **改动点：** 新增 64x64 noise PNG 资源 + 根页面 Stack 内加一个 absolute Image 覆盖层
- **影响范围：** 全局视觉底色变化
- **风险：** ⚠️ 中低——Image 覆盖层必须加 `.hitTestBehavior(HitTestMode.None)` 否则拦截触摸事件；opacity 过高会显脏，过低等于没加
- **缓解：** opacity 控制在 0.02~0.03；覆盖层 zIndex 最低
- **验收：** 侧光/暗光下可感知纸张纹理，正常光下几乎不可见。不影响任何触摸/点击/滑动操作
- **回滚：** 删除 Image 组件 + 删除图片资源

### 2.2 打勾墨水扩散
- **改动点：** ChecklistDetail 中清单列表项模板，checkbox 右侧叠加一个扩散 Row
- **影响范围：** 清单打勾的视觉反馈
- **风险：** ⚠️ 中——扩散层需要用 Stack 包裹列表项内容；如果当前列表项不是 Stack 结构需要包一层；可能影响已有的按压反馈动画时序
- **缓解：** 扩散层 position absolute + hitTestBehavior None，不参与布局；动画延迟避免与按压动画撞车
- **验收：** 打勾时从左到右有极淡绿色墨水扩散（~300ms），不影响打勾响应速度和触摸区域
- **回滚：** 删除扩散层组件

### 2.3 Section Breathing 组间呼吸
- **改动点：** 列表分组间距从统一值改为组间 32vp + 渐隐分隔线 Row
- **影响范围：** 装备库 + 清单列表的分组视觉间距
- **风险：** ⚠️ 低——增大间距会让列表总高度变大，需确认不影响「最后一项 + Tab 安全区」的底部间距计算
- **缓解：** 只改 margin-top of 分组标题，不改组内间距
- **验收：** 分组间视觉上有明显「呼吸」，分组标题上方有极淡渐隐绿线
- **回滚：** 恢复间距值 + 删除分隔线组件

### 2.4 暖琥珀点缀色
- **改动点：** DesignTokens 新增 `ACCENT_WARM = '#C49A6C'` + 3-4 个指定场景使用
- **影响范围：** 倒计时临近时色温 / 空状态底色 / 可能的重量超标提示
- **风险：** ⚠️ 低——纯 additive，不替换现有颜色；需要设计判断「何时从绿转暖」的阈值
- **验收：** App 整体色彩叙事从「纯绿」变为「绿+暖棕」，暖色面积 ≤ 5%
- **回滚：** 删除常量 + 恢复使用处颜色

## Phase 3: 特性级提升（80-150 行，需设计评审）
**状态：** `pending`

### 3.1 首页 Hero 区山脊线装饰
- **改动点：** HomePage hero 区底部加 Shape + Path 矢量山脊线 + 进度指示标记
- **影响范围：** 首页 hero 区视觉
- **风险：** ⚠️ 中——Path commands 的坐标需响应容器宽度变化（折叠屏）；需处理 hero 区折叠时山脊线的收缩逻辑
- **缓解：** Shape 设 width('100%') + viewPort 相对坐标；折叠时用 opacity 淡出即可
- **验收：** hero 区底部有极淡山脊轮廓线，进度标记随完成度左移右移

### 3.2 折叠 Header 视差层
- **改动点：** GearPage header 内部拆为 2 层，各层 translateY 乘以不同系数
- **影响范围：** 装备库页面滚动折叠的视觉效果
- **风险：** ⚠️ 中高——当前折叠逻辑已经比较复杂（scrollOffset 驱动多个属性联动），拆层后需重新调试所有折叠参数
- **缓解：** 先保持功能完全一致，只加视差系数；如果调试成本过高则放弃
- **验收：** 滚动折叠时背景层慢速退出、前景层正常速度，产生深度感

### 3.3 空状态矢量插画
- **改动点：** 装备库空态 / 清单全完成态，用 Shape+Path 绘制极简线条画 + 文案 + 入场动画
- **影响范围：** 空状态页面的情感表达
- **风险：** ⚠️ 低——纯新增组件，不影响有数据时的任何逻辑
- **验收：** 空状态有品牌识别度的线条画 + 温暖文案 + spring 入场

## 明确不做清单
| 方案 | 原因 |
|------|------|
| Tab 切换 translateX | 破坏 HdsTabs + hdsMaterial 沉浸光感体系 |
| 毛玻璃色相偏移 | ArkUI 无此 API 能力 |
| Bento Grid 装备库 | 重构量巨大，与分组折叠/排序/staggered 冲突 |
| 自定义 shader 光效 | ArkUI 不暴露 shader |
| 渐变边框 | Stack 嵌套模拟复杂度 > 收益 |
| 路径动画（点沿曲线运动）| 需手写贝塞尔采样，用 position 平替 |
